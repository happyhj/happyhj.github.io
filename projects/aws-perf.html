Frontend Performance at AWS

Amazon Web Services · SDE II · 2021–2024

Overview

Over three years across Amazon S3 Console, SageMaker Data Wrangler, and SageMaker Canvas, I identified and resolved a series of frontend performance bottlenecks in data-intensive AWS console UIs. Each fix targeted a specific, measurable problem.

Problem → Solution

1. Data Quality Report Browser Freeze

Product: SageMaker Canvas (migrated from Data Wrangler)

Data Wrangler's Data Quality Report page fetched all dataset statistics at once and rendered per-column data quality visualizations using D3 (each creating its own Canvas instance). When the migration to Canvas expanded supported dataset size to 100+ columns, the page attempted to initialize N D3 visualization components simultaneously — freezing the browser.

Solution: Implemented IntersectionObserver-based lazy rendering. Each visualization component only initializes when its containing section scrolls into the viewport. Columns outside the visible area remain unrendered until needed.

Result: Page loads without freezing regardless of column count.

2. Slow DataGrid Response

Product: SageMaker Data Wrangler

The main DataGrid had a 2.8-second response time to user interactions. Profiling revealed excessive re-renders cascading through the component tree on every data change.

Solution: Applied React.memo to visualization components, useCallback for event handlers to stabilize references, and render-on-scroll for DataGrid rows so only visible rows are rendered.

Result: 0.8s response time — 71% reduction.

DataGrid Response Time
0.8s (after) 2.8s (before)

3. Data Insights Initial Render

Product: SageMaker Canvas

After migrating Data Wrangler features into Canvas, the Data Insights page took 10.5 seconds to render and blocked the UI for 75% of that time.

Solution: Systematic React optimization — React.memo on all visualization components to prevent cascade re-renders, useCallback for stable event handler references, and lazy component loading so heavy chart components load on demand.

Result: 0.4s initial render — 97% reduction. UI blocking delay reduced by 75%.

Data Insights Render Time
0.4s (after) 10.5s (before)

4. Bundle Size

Product: SageMaker Canvas

The application bundle had grown to 75MB, impacting cold-start load times for a data-heavy UI that users expect to be responsive.

Solution: Dependency analysis to identify duplicate and unused packages, package consolidation to eliminate redundancy, and tree-shaking audit to ensure dead code elimination.

Result: 44MB — 41% reduction.
